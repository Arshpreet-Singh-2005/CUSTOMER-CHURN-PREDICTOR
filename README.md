## рҹҺҜ Overview

**Customer Churn Predictor** is a full end-to-end machine learning system that identifies customers likely to leave a subscription-based service вҖ” enabling businesses to take **proactive, data-driven retention actions** before revenue is lost.

This project covers the **complete ML lifecycle**: from raw data to a serialized, deployment-ready model вҖ” with a strong emphasis on both **prediction accuracy** and **business interpretability**.

## рҹ“Ҡ Dataset

The dataset contains **real-world telecom customer records** with the following feature categories:

| Category | Features |
|:---|:---|
| рҹ‘Ө Demographics | Gender, SeniorCitizen, Partner, Dependents |
| рҹ“Ў Services | PhoneService, InternetService, StreamingTV, StreamingMovies |
| рҹ“Ӣ Subscription | Contract type, tenure, PaperlessBilling |
| рҹ’і Payments | PaymentMethod, MonthlyCharges, TotalCharges |
| рҹҺҜ **Target** | **`Churn` вҶ’ 1 (left) / 0 (stayed)** |

> **Source:** [Telco Customer Churn вҖ” Kaggle](https://www.kaggle.com/datasets/blastchar/telco-customer-churn)  
> **Size:** ~7,000 customer records В· 21 features

## рҹ”Қ EDA Insights

Key discoveries from exploratory analysis:

```
рҹ“ү  SHORT TENURE  вҶ’  HIGH CHURN RISK
    Customers in their first 12 months churn at nearly 3Г— the rate
    of long-term customers.

рҹ“„  MONTH-TO-MONTH CONTRACTS  вҶ’  MOST VOLATILE
    ~42% churn rate vs <5% for two-year contract holders.

рҹ’°  HIGH MONTHLY CHARGES  вҶ’  ELEVATED RISK
    Customers paying above-average monthly fees show significantly
    higher churn probability.

рҹҢҗ  FIBER OPTIC USERS  вҶ’  SURPRISINGLY HIGH CHURN
    Despite premium service, fiber users churn more вҖ” suggesting
    price sensitivity or unmet expectations.
```

## рҹ”§ Feature Engineering

| Step | Description |
|:---|:---|
| рҹ§№ Missing value handling | Impute or drop based on context |
| рҹ”  Categorical encoding | `LabelEncoder` for binary, `OHE` for multi-class |
| вҡ–пёҸ Feature scaling | `StandardScaler` for numerical stability |
| вңӮпёҸ Redundant feature removal | Drop low-variance & high-correlation columns |
| рҹҺҜ Target encoding | `Churn`: Yes вҶ’ 1, No вҶ’ 0 |

## рҹӨ– Models

| Model | Strength | Use Case |
|:---|:---|:---|

## рҹ“Ҳ Evaluation

Models are compared across five metrics:

```
  Metric          Why It Matters for Churn
  в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
  Accuracy        Overall correctness
  Precision       Are flagged churners actually churning?
  Recall вҳ…        Are we catching all real churners? (key!)
  F1-Score        Balance of precision & recall
  ROC-AUC         Model's ability to rank risk across all thresholds
```

## рҹҡҖ Getting Started

### Prerequisites
- Python 3.8+
- pip

### 1 В· Clone the Repository

```bash
git clone https://github.com/Arshpreet-Singh-2005/CUSTOMER-CHURN-PREDICTOR.git
cd CUSTOMER-CHURN-PREDICTOR
```

### 2 В· Create Virtual Environment

```bash
python -m venv venv

# Windows
venv\Scripts\activate

# Mac / Linux
source venv/bin/activate
```

### 3 В· Install Dependencies

```bash
pip install -r requirements.txt
```

### 4 В· Add the Dataset

Download from [Kaggle](https://www.kaggle.com/datasets/blastchar/telco-customer-churn) and place it at:
```
data/raw/WA_Fn-UseC_-Telco-Customer-Churn.csv
```

### 5 В· Run the Pipeline

```bash
# Clean and preprocess data
python src/data_processing.py

# Train models and save the best one
python src/model_training.py
```

### 6 В· Explore Notebooks (Optional)

```bash
jupyter notebook
```
Open `notebooks/EDA.ipynb` for visual analysis or `notebooks/Modeling.ipynb` for the full training walkthrough.
